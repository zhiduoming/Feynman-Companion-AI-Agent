## 二、各层一句话定位

| 层 | 定位 |
|---|---|
| **L1 前端** | 用户交互入口；Web 优先（桌面 / 小程序为远期扩展） |
| **L2 业务功能** | 考研垂直业务逻辑；调用 L3 完成费曼闭环 |
| **L3 AI 能力** | 大模型 / Agent 编排 / RAG / Prompt 的封装；**领域无关核心引擎主体** |
| **L4 知识处理** | 把原始资料加工为可被 RAG / 推荐使用的数据结构 |
| **L5 数据** | 关系数据、缓存、向量数据、对象存储的统一持久化 |

**冷热分层（数据层）**：
- 热数据：MySQL + Redis（近 30 天高频访问）
- 冷数据：对象存储（原始教材、归档对话）；语音经 ASR 后即刻清除原始音频

## 三、领域无关核心 vs 考研垂直适配

> 本架构的**核心设计思想**：核心引擎与考研垂直内容解耦。
> 首期严格聚焦考研做透；未来扩展到考公 / 技术面试 / K12 等场景时，**只替换右列，核心引擎不动**。

| 层 | 领域无关（可复用） | 考研垂直（领域适配） |
|---|---|---|
| L1 前端 | 对话框、报告样式 | 学科选择、考纲展示 |
| L2 业务 | 费曼对话编排流程 | 学情画像字段、漏洞分类、报告维度 |
| L3 AI | LangGraph 状态机、RAG 流程、Function Calling 协议 | 学科 Prompt 模板、学科 Few-shot 库 |
| L4 知识处理 | 文档解析、切片、Embedding、向量化 | 考纲映射规则、易错点标注 |
| L5 数据 | 关系库 / 向量库 / 缓存结构 | 考研知识点元数据、漏洞类型枚举 |
